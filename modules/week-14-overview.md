## What students usually struggle with
- Students often wait until the end to run `npm run build`, deploy, test keyboard flow, and check Lighthouse. Those are not final-minute tasks; they usually reveal real bugs.
- Treat accessibility, performance, and deployment as one quality pass. Fixing the same issue once is faster than discovering it separately in three different tools.
- Use one written QA note set for both major artifacts this week. The same audit habit should strengthen Project 2 and the Final Project beta, not create two separate scramble sessions.

## Checkpoint question
If your app broke in production today, which one quality check would you run first, and what evidence would that check give you?
